# 建設関連業 公開研修情報DB PoC 観測レポート

**観測日：** 2026-05-24  
**観測者：** karobox  
**リポジトリ：** https://github.com/karobox/learning-opportunities-db  
**公開ページ：** https://karobox.github.io/learning-opportunities-db/

## 概要

建設関連業（土木・建築・測量・地質・補償）を対象に、  
公開研修情報をAIが自動収集・軽量DBとして公開できるか、を試みたPoC記録。

「何が取れたか」と同等に「何が取れなかったか、なぜか」を観測対象としている。

---

## 1. 今回のPoCで確認できたこと

### 1-1. 軽量DB基盤の成立確認

**構成：** GitHub Pages（静的ホスティング）+ CSV + DataTables + PapaParse

- `docs/data/events.csv` を差し替えるだけでページが更新される構造が成立した
- ページ自体はサーバーサイド処理なし（完全静的）
- DataTables によるソート・キーワード検索・ページネーションが動作した
- カスタムフィルタ（分野カテゴリ・開催形式・開催地域・参加資格）が動作した
- 過去イベントの自動除外（`event_date >= today`）が動作した

**判定：** 基盤としては成立。更新コストは低い。

---

### 1-2. AI支援での収集

セッション中、AI（Claude）が以下を担った：

- 候補団体の探索（WebSearch）
- robots.txt の確認
- 個別イベントページからの構造化情報抽出（日時・費用・場所・申込期限）
- CSV への整形・書き込み

**取得実績：** 9件 / 3団体（手動確認込み、1セッション内）

取得に成功した件では、費用・開催日時・申込期限・開催形式・場所がHTML内に明記されており、  
AIが直接読み取れる形で整理されていた。

**判定：** 静的HTMLが整備されている団体については、AI支援での情報抽出は機能した。

---

### 1-3. region_pref 列の設計

開催場所の自由記述（"東京天理ビル3階（東京都千代田区）またはオンライン" 等）とは別に、  
検索用正規化列 `region_pref` を設けた。

- オンライン専用 → `オンライン`
- ハイブリッド → 物理会場の都道府県名（例：`東京都`）
- 全国巡回 → `全国`
- 対面・特定会場 → 都道府県名

フィルタは動作。現時点のデータでは `東京都` と `オンライン` のみ出現している。  
データが増えれば地域分布の観測に使える構造になっている。

## 2. 取得可能だった構造

### 公益社団法人 地盤工学会（jiban.or.jp）

- **CMS：** Joomla（標準的なURL構造）
- **robots.txt：** CMSシステムディレクトリのみ制限。コンテンツページは制限なし
- **情報の整理度：** 高。費用・日時・申込期限・受講時間がHTML内にテキストで明記
- **AI可読性：** 良好。構造が安定しており、ページごとに情報が完結している
- **備考：** 一部講習会は「詳細近日公開」状態。順次追加可能

### 一般財団法人 日本建築センター（bcj.or.jp）

- **CMS：** 独自構造（`/seminar/detail/NNN/` の連番URL）
- **robots.txt：** 管理ページ・アップロードディレクトリのみ制限
- **情報の整理度：** 高。一覧ページと詳細ページの2層構造が安定している
- **AI可読性：** 良好。費用・定員・申込期限・対面/オンライン区別がページ内に記載
- **備考：** 2026年度に11件のラインナップを確認済。うち3件のみ今回取得

### 公益社団法人 日本建築学会（aij.or.jp）

- **CMS：** 独自（`?productId=` 形式）
- **robots.txt：** 管理ページ・会員ページを制限。`claudebot` は明示的に制限なし
- **情報の整理度：** 中程度。費用・場所は記載あり。開始/終了時刻の記載がない講習会もある
- **AI可読性：** 概ね良好。一部フィールドの欠損あり（時刻・受講時間）
- **備考：** `claudebot` を robots.txt で明示的に許可しているのは今回の調査対象中で唯一

## 3. 取得困難だった構造

### PDF依存

**該当：** 一般社団法人 日本補償コンサルタント協会（jcca-net.or.jp）

HTMLページには「〇〇のご案内（お知らせ）」という告知のみ存在し、  
開催日・費用・申込方法はすべてPDFリンク先に格納されていた。  
PDFの取得・解析は今回の構成に含まれていない。

また、費用・会場・日程をPDFから抽出することは、  
著作権的グレーゾーンに接近するリスクもある。

**観測：** 補償分野では、情報の一次公開メディアがPDFである。  
HTMLはあくまで告知の入口に留まっている。

---

**該当：** 国土交通省 関東地方整備局（ktr.mlit.go.jp）

研修一覧ページは存在するが、詳細はPDFファイルへの誘導だった。  
また主要な受講対象が「国・地方公共団体職員」であり、  
一般事業者向け研修とは性格が異なる。

**観測：** 行政機関の研修情報は、対象者が内部に限定されるものが多い。  
「公開研修情報DB」の対象外となる割合が高い。

---

### 文字コード問題（Shift-JIS）

**該当：** 公益社団法人 日本測量協会（jsurvey.jp）

WebFetchツールでの取得時にShift-JIS文字化けが発生した。  
`requests` ライブラリで `response.encoding = 'shift_jis'` を指定すれば解決可能であり、  
環境の限界ではなく実装対応の問題。

**観測：** 測量分野の主要団体サイトが2026年時点でShift-JISを使用している。  
情報公開の形式としてはHTMLが存在するが、エンコーディングの更新が行われていない。

---

### 情報分散

**該当：** 国土交通省本体（mlit.go.jp）

研修・講習会・説明会情報が部署別・事業別に独立したページとして存在しており、  
横断的な一覧ページが存在しない。  
「建築」「道路」「技術調査」「官庁営繕」がそれぞれ別系統で情報を公開している。

**観測：** 国交省の研修情報は「検索して辿り着く」構造になっており、  
自動収集には向いていない。存在は確認できるが、網羅的な取得は困難。

---

**該当：** 公益社団法人 土木学会（jsce.jp / jsce.or.jp）

本部・支部・委員会がそれぞれ独自に研修情報を掲載しており、  
統一された一覧ページが存在しない。  
支部ごとに開催情報が異なり、URLも系統が異なる。

**観測：** 土木学会の研修情報は「構造が統一されていない分散型」。  
個別の支部・委員会ページを順番に確認する手動巡回的なアクセスが必要。

---

### コンテンツの薄さ

**該当：** 一般社団法人 建設コンサルタンツ協会（jcca.or.jp）

robots.txt なし、HTML取得は技術的に問題ないが、  
2026年時点でWebに公開されている研修・セミナー情報が少なかった。  
メインコンテンツはCM/PM関連の委員会活動が中心。

**観測：** 団体によっては、研修情報の一次公開がWeb以外（メール・会報等）のケースがある可能性。

## 4. 今回わかった分野別の情報公開構造の差

| 分野 | 主な団体 | AIからの可読性 | 主な情報媒体 |
|------|----------|----------------|--------------|
| 地盤・基礎 | 地盤工学会 | 高 | 静的HTML（詳細ページ完結型） |
| 建築構造・法規 | 日本建築センター、日本建築学会 | 高 | 静的HTML（連番URL・商品ページ型） |
| 補償業務 | 日本補償コンサルタント協会 | 低 | HTML告知 → PDF本文 |
| 測量 | 日本測量協会 | 中（エンコード対応で改善可） | Shift-JIS HTML |
| 土木全般 | 土木学会 | 中（構造分散） | 支部・委員会別HTML |
| 建設コンサルタント | 建設コンサルタンツ協会 | 未確認（コンテンツ薄） | 不明 |
| 行政（国交省） | 国土交通省・地方整備局 | 低 | 部署別HTML・PDF混在 |

「遅れている／進んでいる」ではなく、  
**それぞれの団体・分野が異なる情報公開文化を持っている**という観察。

## 5. 今回のPoCの限界

### 観測できた範囲

- 公開HTMLから取得できる研修情報の「供給側」の一部
- HTML化・構造化が整備されている団体に偏ったサンプル

### 観測できていない範囲

| 項目 | 理由 |
|------|------|
| 研修の需要・受講者側の動向 | 公開情報に含まれない |
| 会員限定研修の全容 | ログイン不要のみを対象としたため |
| PDF主体の団体の情報 | PDF解析未実施（著作権リスクも含む） |
| 日本語OCR対応 | 未実施 |
| JS描画依存サイト | Playwright等未使用（今回の対象には該当なし） |
| CPDポイント情報 | 設計上の除外対象（認定との混同回避） |
| 国交省本体の全体像 | 分散構造のため横断的取得が困難 |
| 研修内容の詳細・質の評価 | 著作権リスクのため収集対象外 |
| 地域分布の統計的分析 | 現時点9件では分析に不十分 |

### 構造的な留意

収集できた9件は、**情報公開インフラが整備された大規模団体に偏っている**。  
中小規模団体や、Webを主要発信チャネルとしていない団体は捕捉できていない。  
これは「収集できなかった」のではなく「現在の方式で到達できる範囲がここまで」という観測。

## 6. 現時点での結論

### 今の環境でできたこと

- 静的HTML公開の団体に限り、AI支援での情報抽出・CSV整形・DB公開が一連で動いた
- 取得困難な理由（PDF依存・文字化け・情報分散）を分類して記録できた
- 公開ページとしての最低限の機能（検索・フィルタ・日付ソート・地域フィルタ）が動作した

### 追加実装で改善できる範囲

| 改善項目 | 工数 | 効果 |
|----------|------|------|
| Shift-JIS 対応（測量協会） | 小（1行修正） | 測量分野の一部が取得可能になる |
| 日本建築センター残り8件の取得 | 小（同一パターン） | 件数増加 |
| 全地連イベントページの特定 | 中 | 地質分野の追加が可能 |
| PDF テキスト抽出（補償協会） | 大＋著作権検討 | リスク・工数対効果が低い |

### PoCとして十分成立しているか

**「成立している」**

軽量DBとしての構造は動いた。  
取得できた件数（9件）は少ないが、  
「どこまで取れてどこから取れないか」の境界を確認できたことが本来の目的だった。

「小さく正直な研修情報DB」として公開しながら  
「AIから見えるインフラの差」を観測するという二層構造は維持できている。

### 今後の優先順位

1. **日本建築センター残り8件**（今すぐ可・同一方式）
2. **Shift-JIS対応（測量協会）**（実装追加で対応可）
3. **地盤工学会の詳細未公開12件**（順次公開待ち）
4. 土木学会・全地連（調査継続・低優先）
5. 補償・国交省（当面保留）

## 今日はここで止めてよい、という判断理由

- 取得可能な構造（静的HTML型）の主要団体は3つ確認・取得済
- 取得困難な理由が3類型（PDF依存・文字化け・情報分散）に整理された
- JS描画対応・OCRが不要と確認できた（環境の限界ではない）
- 追加取得（BCJ残り8件・測量協会Shift-JIS対応）は「次回以降」として定義できた
- 今日の探索で「PoCとしてどこまで確認できたか」の境界が引けた

**これ以上今日続けても、新しい構造的知見が得られる見込みが低い。**
